# 従業員満足度管理アプリ 要件定義書

**プロジェクト名**: ES Manager（Employee Satisfaction Manager）
**バージョン**: 1.0.0
**作成日**: 2026-03-28
**対象システム**: `axel/employee-estimation`

---

## 1. プロジェクト概要

### 1.1 背景・目的

従業員満足度（Employee Satisfaction）は離職率と強い相関を持ち、組織の持続的な成長に直結する重要な指標である。本アプリケーションは、以下を目的として構築された。

- 四半期ごとに従業員満足度をスコア化し、経時的なトレンドを可視化する
- 管理者が満足度と離職率の相関を把握し、施策の効果を測定できるようにする
- 従業員が簡便にアンケートに回答できる仕組みを提供する

### 1.2 対象ユーザー

| ユーザー区分 | 概要 | 利用画面 |
|---|---|---|
| 従業員 | アンケート回答者。ログイン不要でURLから直接アクセス | `/employee` |
| 管理者 | HR・経営層。満足度トレンドを管理ダッシュボードで閲覧 | `/admin` |

---

## 2. 機能要件

### 2.1 従業員向け機能

| ID | 機能 | 詳細 |
|---|---|---|
| F-01 | アンケート回答 | 10カテゴリ × 5段階評価で回答 |
| F-02 | 回答進捗表示 | 全10問中の回答済み件数をプログレスバーで表示 |
| F-03 | 未回答バリデーション | 全問回答前の送信を防止し、未回答問を自動スクロール |
| F-04 | 送信完了確認 | 送信後に個人スコアをフィードバック表示 |

### 2.2 管理者向け機能

| ID | 機能 | 詳細 |
|---|---|---|
| F-10 | 四半期セレクター | FY25 1Q〜FY26 2Q を複数選択し、表示範囲を絞り込む |
| F-11 | KPIカード表示 | 最新満足度スコア / 前四半期比 / 離職率 / 回答者数 |
| F-12 | 積み上げ棒グラフ | カテゴリ別スコアを四半期横並びで比較 |
| F-13 | 折れ線グラフ | 総合満足度スコアと離職率の推移を重ねて表示 |
| F-14 | カテゴリ別スコアテーブル | 全カテゴリ × 全四半期のスコアマトリクス |
| F-15 | 回答一覧 | 個別回答を四半期フィルターで絞り込み表示 |
| F-16 | サイドバーナビゲーション | ダッシュボード / 回答一覧 へのリンク |

### 2.3 スコアリング

- 各質問: 1〜5点（5段階評価）
- 合計最大点: 50点（10問 × 5点）
- 正規化: `スコア = 合計点 / 50 × 100`（0〜100の整数）
- スコアレベル判定:

| レベル | スコア範囲 | カラー |
|---|---|---|
| 良好 | 80以上 | `#16A34A`（緑） |
| 注意 | 60〜79 | `#D97706`（黄橙） |
| 警告 | 40〜59 | `#EA580C`（橙） |
| 危険 | 39以下 | `#CC0000`（赤） |

## 3. 非機能要件

| 区分 | 要件 |
|---|---|
| デザイン | ガバスコ（governance-tool）と統一されたカラーシステム（紺 `#0D1B3E`） |
| レスポンシブ | PC（lg以上）はサイドバー固定表示、モバイルはハンバーガーメニュー |
| パフォーマンス | Next.js Static Generation により初期表示を高速化 |
| アクセシビリティ | aria-label 付与、キーボード操作対応 |
| 型安全性 | TypeScript strict モードを有効化 |

## 4. 集計サイクル

集計は**四半期ごと**に実施する。

| 期 | 回答期間 | 集計対象四半期 |
|---|---|---|
| FY25 1Q | 2024年4〜6月 | 2024年Q1（4月〜6月） |
| FY25 2Q | 2024年7〜9月 | 2024年Q2（7月〜9月） |
| FY25 3Q | 2024年10〜12月 | 2024年Q3（10月〜12月） |
| FY25 4Q | 2025年1〜3月 | 2024年Q4（1月〜3月） |
| FY26 1Q | 2025年4〜6月 | 2025年Q1（4月〜6月） |
| FY26 2Q | 2025年7〜9月 | 2025年Q2（7月〜9月） |
